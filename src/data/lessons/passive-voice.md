# Passive Voice

The passive voice is used when the focus is on the action, not on who performs it.

## 👉 In active voice:

The developer writes the code. (focus on the doer)

## 👉 In passive voice:

The code is written (by the developer). (focus on the result/action). Structure:

> Object + verb “to be” (conjugated) + past participle (+ by + subject, optional)

- **Active:** She builds the app
- **Passive:** The app is built (by her)

## 💡 Important:

- You can omit the “doer” (by someone) if it’s not important or obvious.
- Very common in technical, formal, or unknown-subject contexts (useful for documentation 👀).

## Examples

| Tense                  | Examples (EN → ES)                                                                                                                                                                                                                               |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Simple Present**     | 1. The code is reviewed every day. → El código es revisado todos los días. <br> 2. The app is deployed automatically. → La aplicación es desplegada automáticamente. <br> 3. Emails are sent in bulk. → Los correos son enviados en masa.        |
| **Simple Past**        | 1. The bug was fixed yesterday. → El error fue corregido ayer. <br> 2. The feature was released last week. → La funcionalidad fue lanzada la semana pasada. <br> 3. The server was restarted at night. → El servidor fue reiniciado en la noche. |
| **Present Continuous** | 1. The system is being updated right now. → El sistema está siendo actualizado ahora mismo. <br> 2. The data is being processed. → Los datos están siendo procesados. <br> 3. The UI is being redesigned. → La interfaz está siendo rediseñada.  |
| **Present Perfect**    | 1. The project has been completed. → El proyecto ha sido completado. <br> 2. The changes have been applied. → Los cambios han sido aplicados. <br> 3. The issue has been resolved. → El problema ha sido resuelto.                               |
| **Future (will)**      | 1. The app will be launched tomorrow. → La aplicación será lanzada mañana. <br> 2. The feature will be added soon. → La funcionalidad será agregada pronto. <br> 3. The bug will be fixed. → El error será corregido.                            |
| **Modal Verbs**        | 1. The code can be improved. → El código puede ser mejorado. <br> 2. The task must be completed today. → La tarea debe ser completada hoy. <br> 3. The API should be documented. → La API debería ser documentada.                               |

## When to use passive voice (practical tips)
- ✅ When the doer is unknown:
  - `The data was stolen.`
- ✅ When the action/result is more important:
  - `The app was deployed successfully.`
- ✅ In technical writing / documentation (very common in your dev context):
  - `The request is validated before processing.`
